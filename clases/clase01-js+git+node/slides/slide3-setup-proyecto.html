📁 PASO 1: Crear Repositorio GitHub

🎯 Objetivo: Crear tu repositorio personal siguiendo el método recomendado

🌐 En GitHub.com:

1. Ir a github.com 2. Click en "New Repository" 3. Nombre: mi-api-[tu-eleccion] Ejemplos: • mi-api-blog • mi-api-restaurante • mi-api-elearning • mi-api-ecommerce 4. ✅ MARCAR "Add a README file" 5. ✅ MARCAR "Add .gitignore: Node" 6. Click "Create repository"
✅ ¿Por qué este método?
  • README.md ya está creado
  • .gitignore configurado para Node.js
  • Primer commit automático
  • Remote origin ya configurado

💻 En tu computadora:

# Clonar el repositorio git clone https://github.com/tu-usuario/mi-api-blog.git # Entrar a la carpeta cd mi-api-blog # Verificar que todo está bien git status git log --oneline
🔍 Verificar que funciona:
  • git status → "working tree clean"
  • git log → muestra el commit inicial
  • ls → ve README.md y .gitignore
⏰ Tiempo: 10 minutos para crear y clonar tu repositorio

📂 PASO 2: Estructura Inicial del Proyecto

🎯 Objetivo: Crear los archivos básicos para tu API

📁 Estructura que vas a crear:

mi-api-[tipo]/ ├── README.md ← Ya existe ├── .gitignore ← Ya existe ├── server.js ← Crear ahora ├── package.json ← Crear en Clase 2 └── (se irá llenando cada clase)

📝 Crear server.js:

// server.js console.log('🚀 Hola! Soy tu API personal'); console.log('📅 Fecha:', new Date().toLocaleDateString()); const args = process.argv.slice(2); if (args.length === 0) { console.log('💡 Prueba: node server.js tu-nombre'); } else { const nombre = args[0]; console.log(`👋 Hola ${nombre}! Tu API está funcionando`); } console.log('✅ Script ejecutado correctamente');

🧪 Probar el script:

# Probar sin argumentos node server.js # Probar con tu nombre node server.js Ana
✅ Resultado esperado:
  • Sin argumentos: muestra mensaje de ayuda
  • Con nombre: saluda personalmente
  • Muestra fecha actual
  • Sin errores en terminal
⚠️ Solo cambiar:
  • Nada más por ahora
  • Solo verificar que funciona
  • Es un script temporal para practicar
⏰ Tiempo: 10 minutos para crear y probar server.js

💾 PASO 3: Primer Commit

🎯 Objetivo: Guardar tu progreso en Git y subir a GitHub

🔍 Verificar estado:

# Ver qué archivos hay ls -la # Ver estado de Git git status # Resultado esperado: # Changes not staged for commit: # modified: README.md (si lo editaste) # Untracked files: # server.js

➕ Agregar archivos:

# Agregar todos los cambios git add . # Verificar que se agregaron git status # Resultado esperado: # Changes to be committed: # modified: README.md # new file: server.js

💾 Hacer commit:

# Hacer commit descriptivo git commit -m "Clase 1: Setup inicial + primer script Node.js - Agregado server.js con información del proyecto - Script funciona con 'node server.js' - Comando especial 'node server.js plan' - Configuración inicial completada" # Subir a GitHub git push origin main
✅ Verificar en GitHub:
  • Refrescar tu repositorio en GitHub
  • Deberías ver server.js
  • El commit aparece en el historial
  • README.md actualizado (si lo editaste)
⏰ Tiempo: 5 minutos para commit y push
